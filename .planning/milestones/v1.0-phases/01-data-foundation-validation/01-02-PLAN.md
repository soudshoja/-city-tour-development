<feature>
  <name>Bulk Upload Validation Service</name>
  <files>app/Services/BulkUploadValidationService.php, tests/Feature/BulkUploadValidationTest.php</files>
  <behavior>
    The service accepts parsed Excel data and a context (company_id, agent_id) and returns structured validation results.

    **Method: `validateHeaders(array $headers): array`**
    - Input: array of header strings from first row
    - Expected headers: ['task_id', 'client_mobile', 'supplier_name', 'task_type', 'task_status', 'invoice_date', 'currency', 'notes']
    - Required headers: task_id, client_mobile, supplier_name, task_type (others optional)
    - Output: `['valid' => bool, 'missing' => [...], 'extra' => [...]]`
    - Cases:
      - All correct headers -> `['valid' => true, 'missing' => [], 'extra' => []]`
      - Missing 'task_id' -> `['valid' => false, 'missing' => ['task_id'], 'extra' => []]`
      - Extra 'foo' column -> `['valid' => true, 'missing' => [], 'extra' => ['foo']]` (extra cols allowed, just warned)

    **Method: `validateRow(array $row, int $rowNumber, int $companyId): array`**
    - Input: associative array of row data, 1-indexed row number, company context
    - Output: `['status' => 'valid'|'error'|'flagged', 'errors' => [...], 'flag_reason' => null|string, 'matched' => ['client_id' => ?int, 'task_id' => ?int, 'supplier_id' => ?int]]`
    - Validation rules:
      1. task_id: required, integer, must exist in tasks table with matching company_id
      2. task_id: must NOT already appear in invoice_details (not already invoiced)
      3. client_mobile: required, string — lookup Client where company_id + phone match
      4. client_mobile: if no match found, status='flagged', flag_reason='unknown_client' (NOT an error)
      5. supplier_name: required — lookup Supplier by name (case-insensitive via LIKE or whereRaw LOWER)
      6. supplier_name: if not found, error
      7. task_type: required, must be one of the 12 enum values
      8. task_status: optional, if provided must be valid enum
      9. invoice_date: optional, if provided must be valid date format (Y-m-d)
      10. currency: optional, if provided must be 3-letter code (e.g., KWD, USD)
    - Cases:
      - Valid row with known client -> status 'valid', all matched IDs populated
      - Missing task_id -> status 'error', errors: ['Row 3: task_id is required']
      - Unknown client mobile -> status 'flagged', flag_reason: 'unknown_client', NOT in errors
      - Task already invoiced -> status 'error', errors: ['Row 3: task_id 456 is already invoiced']
      - Invalid task_type 'boat' -> status 'error', errors: ['Row 3: task_type "boat" is invalid. Must be one of: flight, hotel, ...']
      - Multiple errors on one row -> all collected in errors array

    **Method: `validateAll(array $rows, int $companyId): array`**
    - Input: array of row arrays (parsed from Excel), company context
    - Output: `['total' => int, 'valid' => int, 'errors' => int, 'flagged' => int, 'rows' => [...results from validateRow]]`
    - Iterates all rows, calls validateRow for each, aggregates counts
  </behavior>
  <implementation>
    Create `app/Services/BulkUploadValidationService.php`:
    - Constructor: no dependencies (uses Eloquent models directly)
    - Constants: EXPECTED_HEADERS, REQUIRED_HEADERS, VALID_TASK_TYPES, VALID_TASK_STATUSES, VALID_CURRENCIES (at minimum KWD, USD, EUR, GBP, SAR, AED, BHD, OMR, QAR)
    - Each validation method is pure logic that queries the database
    - Error messages always include row number and field name: "Row {N}: {field} {message}"
    - Client matching: `Client::where('company_id', $companyId)->where('phone', $mobile)->first()`
    - Task existence: `Task::where('id', $taskId)->where('company_id', $companyId)->first()`
    - Task already invoiced: `InvoiceDetail::where('task_id', $taskId)->exists()`
    - Supplier lookup: `Supplier::whereRaw('LOWER(name) = ?', [strtolower($supplierName)])->first()`

    Tests should use `RefreshDatabase` trait and create test data using factories or direct model creation.
    Test each validation rule independently plus a full integration test with mixed valid/error/flagged rows.
  </implementation>
</feature>
