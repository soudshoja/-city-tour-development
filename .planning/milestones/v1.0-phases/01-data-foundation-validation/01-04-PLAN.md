<objective>
Create the error report Excel export so agents can download a clear breakdown of validation failures for large uploads.

Purpose: When an upload has many validation errors, reviewing them in the browser is impractical. This plan creates a downloadable Excel report that lists every error and flagged row with the original data, row number, error messages, and flag reasons. Color-coded for quick scanning. Covers UPLOAD-06.

Output: BulkUploadErrorReportExport class, downloadErrorReport method on BulkInvoiceController, GET route.
</objective>

  <action>
    Create `app/Exports/BulkUploadErrorReportExport.php`:

    Implement: `Maatwebsite\Excel\Concerns\FromArray`, `WithHeadings`, `WithStyles`, `ShouldAutoSize`, `WithEvents`

    Constructor: accepts `BulkUpload $bulkUpload`

    **headings()**: Return column headers:
    `['Row #', 'Status', 'Task ID', 'Client Mobile', 'Supplier Name', 'Task Type', 'Task Status', 'Invoice Date', 'Currency', 'Notes', 'Errors', 'Flag Reason']`

    **array()**: Query `$this->bulkUpload->rows()->whereIn('status', ['error', 'flagged'])->orderBy('row_number')->get()` and map each to an array:
    ```php
    return $rows->map(function ($row) {
        $rawData = $row->raw_data ?? [];
        return [
            $row->row_number,
            strtoupper($row->status),
            $rawData['task_id'] ?? '',
            $rawData['client_mobile'] ?? '',
            $rawData['supplier_name'] ?? '',
            $rawData['task_type'] ?? '',
            $rawData['task_status'] ?? '',
            $rawData['invoice_date'] ?? '',
            $rawData['currency'] ?? '',
            $rawData['notes'] ?? '',
            is_array($row->errors) ? implode('; ', $row->errors) : ($row->errors ?? ''),
            $row->flag_reason ?? '',
        ];
    })->toArray();
    ```

    **styles(Worksheet $sheet)**: Style the header row:
    - Row 1: bold, white text, dark blue background (#4472C4), centered
    - Column A (Row #): width 8
    - Column B (Status): width 10
    - Column K (Errors): width 50 (errors can be long)

    **registerEvents()**: Use `AfterSheet` event for conditional row coloring:
    - Iterate data rows (starting row 2)
    - If column B = 'ERROR': fill row with light red (#FFC7CE)
    - If column B = 'FLAGGED': fill row with light yellow (#FFEB9C)
    - This uses `$event->sheet->getStyle("A{$row}:L{$row}")->applyFromArray([...])`

    Also add a summary row at the top or bottom:
    - After the last data row, add an empty row then a summary: "Total errors: X, Total flagged: Y"
    - Use `$this->bulkUpload->error_rows` and `$this->bulkUpload->flagged_rows` for counts

  <action>
    **BulkInvoiceController::downloadErrorReport()** — Add method to existing controller:

    Method signature: `public function downloadErrorReport(int $id): \Symfony\Component\HttpFoundation\BinaryFileResponse`

    Logic:
    1. Get authenticated user and company_id: `$companyId = getCompanyId(Auth::user())`
    2. Find the BulkUpload: `$bulkUpload = BulkUpload::where('id', $id)->where('company_id', $companyId)->firstOrFail()`
       - The `where('company_id', $companyId)` ensures multi-tenant isolation — agent cannot download reports for other companies
       - `firstOrFail()` returns 404 if not found or not their company
    3. Check if there are errors to report: if `$bulkUpload->error_rows === 0 && $bulkUpload->flagged_rows === 0`, return redirect back with message "No errors or flagged rows to export."
    4. Generate filename: `'error-report-' . Str::slug($bulkUpload->original_filename, '-') . '-' . $bulkUpload->id . '.xlsx'`
    5. Return: `Excel::download(new BulkUploadErrorReportExport($bulkUpload), $filename)`

    **Imports to add** to BulkInvoiceController (if not already present from 01-03):
    - `use App\Exports\BulkUploadErrorReportExport;`
    - `use Illuminate\Support\Str;`

    **Route registration** in `routes/web.php`:
    - Add to the existing `bulk-invoices` route group:
      ```php
      Route::get('/{id}/error-report', [BulkInvoiceController::class, 'downloadErrorReport'])->name('error-report');
      ```
    - This goes alongside the existing template and upload routes

    Error handling: wrap in try/catch. On exception, log and return 500.
