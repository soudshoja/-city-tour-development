<objective>
Build the admin API for managing per-company DOTW credentials: a POST (upsert) and GET endpoint at `/api/admin/companies/{companyId}/dotw-credentials`, with a FormRequest enforcing field-level validation so that missing fields return specific error messages (not generic 500s), and strict response sanitization so encrypted credentials are never exposed.

Purpose: Operators must be able to configure DOTW credentials per company through the API. This is the only write path for the credential table created in Plan 01. Without it, credentials can only be seeded manually in the database, which is not operationally viable.

Output:
- StoreDotwCredentialRequest with per-field validation messages
- DotwCredentialController with upsert (store) and show actions
- Two routes registered in api.php under /admin/companies/{companyId}/dotw-credentials
</objective>

    **Create `app/Http/Controllers/Admin/DotwCredentialController.php`**:

    Namespace: `App\Http\Controllers\Admin`
    Extends: `App\Http\Controllers\Controller`

    Import:
    ```php
    use App\Models\Company;
    use App\Models\CompanyDotwCredential;
    use App\Http\Requests\StoreDotwCredentialRequest;
    use Illuminate\Http\JsonResponse;
    ```

    Two methods:

    **`store(StoreDotwCredentialRequest $request, int $companyId): JsonResponse`**
    ```php
    // Verify the company exists (404 if not)
    $company = Company::findOrFail($companyId);

    // Upsert — update existing row or create new one (unique constraint on company_id)
    $credential = CompanyDotwCredential::updateOrCreate(
        ['company_id' => $companyId],
        [
            'dotw_username'     => $request->input('dotw_username'),
            'dotw_password'     => $request->input('dotw_password'),
            'dotw_company_code' => $request->input('dotw_company_code'),
            'markup_percent'    => $request->input('markup_percent', 20.00),
            'is_active'         => true,
        ]
    );

    return response()->json([
        'success'    => true,
        'message'    => 'DOTW credentials saved successfully',
        'company_id' => $companyId,
        'markup_percent' => $credential->markup_percent,
        // Note: dotw_username and dotw_password are intentionally NOT returned
        // The $hidden property on CompanyDotwCredential excludes them from serialization
    ], 200);
    ```

    **`show(int $companyId): JsonResponse`**
    ```php
    $company = Company::findOrFail($companyId);

    $credential = CompanyDotwCredential::where('company_id', $companyId)
        ->where('is_active', true)
        ->first();

    if (!$credential) {
        return response()->json([
            'success'    => false,
            'configured' => false,
            'company_id' => $companyId,
            'message'    => 'DOTW credentials not configured for this company',
        ], 404);
    }

    return response()->json([
        'success'         => true,
        'configured'      => true,
        'company_id'      => $companyId,
        'dotw_company_code' => $credential->dotw_company_code,
        'markup_percent'  => $credential->markup_percent,
        'is_active'       => $credential->is_active,
        'created_at'      => $credential->created_at,
        'updated_at'      => $credential->updated_at,
        // dotw_username and dotw_password are NOT returned — hidden at model level
    ]);
    ```

    Add PHPDoc blocks on both methods. The `show` method intentionally omits credentials from the response — document this with an inline comment.
