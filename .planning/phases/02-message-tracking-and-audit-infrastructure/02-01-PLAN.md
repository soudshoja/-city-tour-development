  <action>
Create `app/Services/DotwAuditService.php`. This service is the single point through which all DOTW audit logs are written. Its two responsibilities are: (1) sanitize payloads before persisting, and (2) write to `dotw_audit_logs` via `DotwAuditLog::log()`.

Class structure:

```
namespace App\Services;

use App\Models\DotwAuditLog;
use Illuminate\Support\Facades\Log;

class DotwAuditService
```

**Method: `log(string $operationType, array $request, array $response, ?string $resayilMessageId = null, ?string $resayilQuoteId = null, ?int $companyId = null): DotwAuditLog`**

This is the primary public method. Steps:
1. Call `$this->sanitizePayload($request)` on the request array.
2. Call `$this->sanitizePayload($response)` on the response array.
3. Call `DotwAuditLog::log([...])` with all six fields.
4. Return the created model instance.
5. Wrap in try/catch — if logging fails, write to `Log::channel('dotw')->error(...)` but do NOT rethrow (audit failure must never break the operation).

**Method: `private function sanitizePayload(array $payload): array`**

Recursively strip any key from the payload (at any nesting depth) whose name matches any of these case-insensitive patterns: `password`, `dotw_password`, `dotw_username`, `username`, `md5`, `secret`, `token`, `authorization`, `credit_card`, `card_number`, `cvv`, `passport_number`.

Use `array_walk_recursive` or a recursive helper. Replacement value for stripped keys: `'[REDACTED]'`.

