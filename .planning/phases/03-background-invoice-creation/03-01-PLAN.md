  <action>
    Create `app/Jobs/CreateBulkInvoicesJob.php` implementing `ShouldQueue` with the following structure:

    **Class setup:**
    - Use traits: Dispatchable, InteractsWithQueue, Queueable, SerializesModels
    - Properties: `$tries = 3`, `$backoff = [10, 30, 60]`, `$timeout = 300`
    - Constructor accepts `public int $bulkUploadId`

    **handle() method:**
    1. Load BulkUpload with eager loading: `->with('rows.client', 'rows.supplier', 'rows.task')->findOrFail($this->bulkUploadId)`
    2. Set log context: `Log::withContext(['bulk_upload_id' => $this->bulkUploadId, 'company_id' => $bulkUpload->company_id])`
    3. Log start: filename, valid_rows count
    4. Wrap everything in `DB::transaction(function () { ... })`

    **Inside the transaction:**
    a. Get valid rows: `$bulkUpload->rows()->where('status', 'valid')->get()`
    b. Group by composite key using SAME logic as BulkInvoiceController::preview() (lines 275-280):
       ```php
       $invoiceGroups = $validRows->groupBy(function ($row) {
           $clientId = $row->client_id;
           $invoiceDate = $row->raw_data['invoice_date'] ?? date('Y-m-d');
           return "{$clientId}_{$invoiceDate}";
       });
       ```
    c. For each group:
       - Call `$this->generateInvoiceNumber($bulkUpload->company_id)` to get next invoice number with pessimistic lock
       - Get first row for shared metadata (client, agent)
       - Calculate subTotal: `$rows->sum(fn($r) => (float)($r->raw_data['task_price'] ?? $r->task->total))`
       - Create Invoice using SAME fields as InvoiceController::store() (lines 1222-1233):
         ```php
         Invoice::create([
             'invoice_number' => $invoiceNumber,
             'client_id' => (int) $clientId,
             'agent_id' => $bulkUpload->agent_id,
             'currency' => $firstRow->raw_data['currency'] ?? 'KWD',
             'sub_amount' => $subTotal,
             'amount' => $subTotal,
             'invoice_date' => $invoiceDate,
             'status' => 'unpaid',  // Use 'unpaid' matching InvoiceStatus::UNPAID
         ]);
         ```
       - For each row in the group:
         - Check duplicate task: `InvoiceDetail::where('task_id', $row->task_id)->whereHas('invoice', fn($q) => $q->whereNotIn('status', ['refunded', 'paid by refund']))->exists()`
         - If duplicate found: throw Exception with message identifying the task_id and existing invoice number (transaction rolls back ALL invoices)
         - Create InvoiceDetail using SAME fields as InvoiceController::store() (lines 1257-1270):
           ```php
           InvoiceDetail::create([
               'invoice_id' => $invoice->id,
               'invoice_number' => $invoiceNumber,
               'task_id' => $row->task_id,
               'task_description' => $row->raw_data['task_description'] ?? $row->task->reference ?? '',
               'task_price' => (float)($row->raw_data['task_price'] ?? $row->task->total),
               'supplier_price' => (float)($row->task->price ?? $row->task->total ?? 0),
               'markup_price' => $taskPrice - $supplierPrice,
               'profit' => $taskPrice - $supplierPrice,
               'paid' => false,
           ]);
           ```
       - Collect invoice ID
       - Log each created invoice: invoice_id, invoice_number, client_id, task_count
    d. After loop: Update BulkUpload status and invoice_ids:
       ```php
       $bulkUpload->update([
           'status' => 'completed',
           'invoice_ids' => $invoiceIds,
       ]);
       ```
    5. Log completion: invoices_created count, invoice_ids array

    **generateInvoiceNumber(int $companyId): string method:**
    - Use `lockForUpdate()` on InvoiceSequence to prevent race conditions (NOT the existing pattern without locking from InvoiceController)
    - `InvoiceSequence::where('company_id', $companyId)->lockForUpdate()->first()`
    - If no sequence exists: `InvoiceSequence::create(['company_id' => $companyId, 'current_sequence' => 1])`
    - Format: `sprintf('INV-%s-%05d', now()->year, $sequence->current_sequence)` -- matches existing `generateInvoiceNumber()` in InvoiceController (line 1977-1981)
    - Increment: `$sequence->increment('current_sequence')`
    - Return the invoice number string

    **failed(Throwable $exception) method:**
    - Log the permanent failure with bulk_upload_id, exception message, and stack trace
    - Update BulkUpload status to 'failed' with error details in error_summary:
      ```php
      BulkUpload::where('id', $this->bulkUploadId)->update([
          'status' => 'failed',
          'error_summary' => json_encode([
              'job_failure' => $exception->getMessage(),
              'failed_at' => now()->toDateTimeString(),
          ]),
      ]);
      ```
      Note: Use `json_encode()` directly since this update bypasses the model cast (using query builder update, not Eloquent save).

    **Important implementation notes:**
    - Use `'unpaid'` for invoice status (matching InvoiceStatus::UNPAID enum value and existing InvoiceController pattern)
    - Do NOT dispatch job inside a transaction -- the controller handles dispatch with `afterCommit()` (Plan 02)
    - The duplicate task check throws an exception that rolls back ALL invoices in the transaction -- this is intentional for atomicity (all or nothing)
    - Import all required classes: BulkUpload, BulkUploadRow, Invoice, InvoiceDetail, InvoiceSequence, DB, Log, plus queue traits
