```

DOTW_COMPANY_CODE      | (none)   | Yes      | DOTW company code
```

**Section 4: New Installation Guide (MOD-07)**
Step-by-step, numbered list:

1. Copy all files listed in Section 2 to the target Laravel installation.
2. Add environment variables from Section 3 to `.env`.
3. Run migrations:
   ```bash
   php artisan migrate
   ```
   This creates: `company_dotw_credentials`, `dotw_audit_logs`, `dotw_prebooks`, `dotw_rooms`, `dotw_bookings` tables.
   Prerequisites: The `companies` table must exist (`company_dotw_credentials` has a FK to it).
4. Register the GraphQL schema — add to `graphql/schema.graphql`:
   ```
   #import dotw.graphql
   ```
5. Register GraphQL route middleware — add to `config/lighthouse.php` under `route.middleware`:
   ```php
   \App\GraphQL\Middleware\ResayilContextMiddleware::class,
   \App\GraphQL\Middleware\DotwTraceMiddleware::class,
   ```
6. Register HTTP middleware alias — add to `bootstrap/app.php`:
   ```php
   'dotw_audit_access' => \App\Http\Middleware\DotwAuditAccess::class,
   ```
7. Register admin route — add to `routes/web.php`:
   ```php
   Route::middleware(['auth', 'dotw_audit_access'])
       ->prefix('admin/dotw')
       ->name('admin.dotw.')
       ->group(function () {
           Route::get('audit-logs', \App\Http\Livewire\Admin\DotwAuditLogIndex::class)
               ->name('audit-logs');
       });
   ```
8. Configure DOTW logging — add to `config/logging.php` channels array:
   ```php
   'dotw' => [
       'driver' => 'daily',
       'path' => storage_path('logs/dotw/dotw.log'),
       'level' => 'debug',
       'days' => 14,
   ],
   ```
9. Clear caches: `php artisan optimize:clear`

**Section 5: Per-Company Credential Setup (B2B Path)**
Explain that in production, each company has its own DOTW credentials stored encrypted in the database:
- Use the `company_dotw_credentials` table via `CompanyDotwCredential` model
- Credentials are stored encrypted using Laravel's `Crypt::encrypt()` — never in plaintext
- The `DotwService` constructor accepts `?int $companyId` — when passed, it loads DB credentials; when null, it falls back to env vars
- Store credentials: `CompanyDotwCredential::updateOrCreate(['company_id' => $id], ['dotw_username' => '...', 'dotw_password' => '...', 'dotw_company_code' => '...', 'markup_percent' => 20])`
- Each company can have a custom `markup_percent` (default 20%)

**Section 6: B2B API Consumer Guide (B2B-05)**
This section is for external B2B partners consuming the GraphQL API.

6.1 — Request Headers (required for all operations):
```
X-Company-ID: {company_id}         (integer — identifies the company making the request)
X-Resayil-Message-ID: {message_id} (string — WhatsApp/Resayil message ID for traceability)
X-Resayil-Quote-ID: {quote_id}     (string — optional, for quoted-message context)
```
Note: `X-Company-ID` is used to resolve per-company DOTW credentials. If omitted or invalid, operations fail with `CREDENTIALS_NOT_CONFIGURED`.

6.2 — Response envelope (all operations return this structure):
```json
{
  "data": {
    "operationName": {
      "success": true,
      "error": null,
      "meta": {
        "trace_id": "uuid",
        "timestamp": "2026-02-21T10:00:00Z",
        "company_id": 1,
        "request_id": "uuid"
      },
      "cached": false,
      "data": { ... }
    }
  }
}
```

6.3 — Error response structure:
```json
{
  "success": false,
  "error": {
    "error_code": "CREDENTIALS_NOT_CONFIGURED",
    "error_message": "DOTW credentials not configured for this company",
    "error_details": "Technical detail for debugging",
    "action": "RECONFIGURE_CREDENTIALS"
  }
}
```
Error code values: `CREDENTIALS_NOT_CONFIGURED`, `CREDENTIALS_INVALID`, `ALLOCATION_EXPIRED`, `RATE_UNAVAILABLE`, `HOTEL_SOLD_OUT`, `PASSENGER_VALIDATION_FAILED`, `API_TIMEOUT`, `API_ERROR`, `CIRCUIT_BREAKER_OPEN`, `VALIDATION_ERROR`, `INTERNAL_ERROR`

6.4 — Example Queries (one per operation):

**getCities:**
```graphql
query GetCities {
  getCities(country_code: "AE") {

    meta { trace_id }
```

```graphql
}
```

6.5 — Booking Flow Sequence:
```
1. getCities(country_code) → city codes
2. searchHotels(destination, dates, rooms) → hotel_code + cheapest_rates
3. getRoomRates(hotel_code, dates, rooms) → allocation_details + detailed rates
4. blockRates(hotel_code, dates, rooms, room_type, rate_basis, allocation_details) → prebook_key (3-min window)
5. createPreBooking(prebook_key, passengers) → confirmation_code
```
