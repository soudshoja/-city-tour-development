```php
/**
 * Delete a saved (unconfirmed) itinerary
 *
 * Removes an itinerary created by savebooking that has not yet been confirmed
 * via bookitinerary. Only for APR flow — cannot delete confirmed bookings.
 *
 * @param  string  $itineraryCode  Code returned from saveBooking
 * @return array  Deletion result
 *
 * @throws Exception  If deletion fails
 */
public function deleteItinerary(string $itineraryCode): array
{
    $this->logger->info('DOTW deleteItinerary request initiated', [
        'itinerary_code' => $itineraryCode,
    ]);

    $body = sprintf(
        '<bookingDetails>
      <bookingCode>%s</bookingCode>
    </bookingDetails>',
        htmlspecialchars($itineraryCode)
    );
    $xml = $this->wrapRequest('deleteitinerary', $body);

    $response = $this->post($xml);

    if ((string) $response->successful !== 'TRUE') {
        $errorCode = (string) ($response->request->error->code ?? 'UNKNOWN');
        $errorDetails = (string) ($response->request->error->details ?? 'Unknown error');

        $this->logger->error('DOTW deleteItinerary error', [
            'error_code' => $errorCode,
            'error_details' => $errorDetails,
            'itinerary_code' => $itineraryCode,
        ]);

        throw new Exception("DOTW deleteItinerary error [{$errorCode}]: {$errorDetails}");
    }

    $this->logger->info('DOTW deleteItinerary successful', [
        'itinerary_code' => $itineraryCode,
    ]);

    return ['deleted' => true, 'itineraryCode' => $itineraryCode];
}
```

**DotwCheckCancellation.php** (`App\GraphQL\Queries`):
- Resolves company_id from `auth()->user()?->company?->id`
- Calls `(new DotwService($companyId))->cancelBooking(['bookingCode' => $bookingCode, 'confirm' => 'no', 'bookingType' => '1'])`
- The service already handles both confirm=no and confirm=yes — use the same `cancelBooking()` method
- Parse the response: `$charge = (float) ($result['charge'] ?? 0)` (the `parseCancellation()` in DotwService returns a `charge` key — verify by reading the `parseCancellation` method)
- Return `CancellationChargeData { booking_code: $bookingCode, charge: $charge, currency: '', is_outside_deadline: $charge === 0.0 }`
- Note: DOTW does not always return currency in cancel query response — default to empty string

**DotwCancelBooking.php** (`App\GraphQL\Mutations`):
- Receives `booking_code: String!` and `penalty_applied: Float!`
- VALID-02 enforcement: Load the DotwBooking record by confirmation_code. Check `$booking->hotel_details['is_apr'] ?? false`. If true, return error: `errorResponse('VALIDATION_ERROR', 'APR (non-refundable) bookings cannot be cancelled. Cancel and amend are not available for advance purchase rates.', 'NONE')`
- If not APR, call `(new DotwService($companyId))->cancelBooking(['bookingCode' => $bookingCode, 'confirm' => 'yes', 'penaltyApplied' => $penaltyApplied, 'bookingType' => '1'])`
- Parse result: `productsLeft = (int) ($result['productsLeftOnItinerary'] ?? 0)`
- Return `CancelBookingData { booking_code, cancelled: true, penalty_applied: $penaltyApplied, products_left_on_itinerary: $productsLeft }`
- If DotwBooking record is not found by confirmation_code, still attempt the cancel (the booking may have been made before v2.0) — skip the APR check if not found

**DotwDeleteItinerary.php** (`App\GraphQL\Mutations`):
- Receives `itinerary_code: String!`
- Calls `(new DotwService($companyId))->deleteItinerary($itineraryCode)`
- Returns `DeleteItineraryData { itinerary_code: $itineraryCode, deleted: true }`
- Error handling: catch Exception → API_ERROR

All three resolvers need a private `buildMeta(int $companyId): array` helper and private `errorResponse(...)` helper. Copy these verbatim from DotwCreatePreBooking — the pattern is identical across all resolvers.
