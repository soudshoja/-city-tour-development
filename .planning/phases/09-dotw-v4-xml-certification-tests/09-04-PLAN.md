**VALID-01 Fix: validateBlockingStatus() reads status as attribute, but DOTW returns it as a child element.**

Current code (line ~1291):
```php
$status = (string) ($rateBasis['status'] ?? 'unchecked');
```

This reads a `status` XML attribute (`<rateBasis status="checked">`). But the DOTW response has `<status>checked</status>` as a child element. The DotwCertify test (confirmed working) uses `(string)($blockRateBasis->status ?? '')` as an element access.

Fix: change the one line to:
```php
$status = (string) ($rateBasis->status ?? 'unchecked');
```

Also update the PHPDoc on `validateBlockingStatus()` to explain: "Validates that each room's rateBasis `<status>` child element equals `checked` after blocking getRooms. DOTW returns status as an XML element, not an attribute."

**VALID-03: Changed occupancy support in buildConfirmRoomsXml() and the rooms array protocol.**

The service's `buildConfirmRoomsXml()` already handles `$room['adultsCode']`, `$room['actualAdults']`, `$room['children']`, `$room['actualChildren']`. It does NOT handle `$room['extraBed']`.

Add `extraBed` field support to `buildConfirmRoomsXml()`:

In the sprintf for each room element, after the `<beddingPreference>` line and before `<passengersDetails>`, add:
```php
$extraBedXml = '';
if (isset($room['extraBed']) && (int)$room['extraBed'] > 0) {
    $extraBedXml = sprintf('<extraBed>%d</extraBed>', (int)$room['extraBed']);
}
```

Then include `$extraBedXml` in the sprintf output between `<beddingPreference>` and `$passengersXml`.

Also update the PHPDoc on `buildConfirmRoomsXml()` to document the `changedOccupancy` contract:
```
 * When a rate has changedOccupancy, the caller MUST pass:
 *   - 'adultsCode'    => from validForOccupancy/adults
 *   - 'children'      => from validForOccupancy/children + childrenAges
 *   - 'extraBed'      => from validForOccupancy/extraBed (0 if not present)
 *   - 'actualAdults'  => original search value
 *   - 'actualChildren' => original search children array
```

**VALID-04 Gzip verification:** Read the existing `post()` private method in DotwService. It already has:
```php
Http::withHeaders(['Accept-Encoding' => 'gzip', ...])->withOptions(['decode_content' => true])
```

This is correct and complete. No code change needed. Add a comment above the Accept-Encoding header line:
```php
// VALID-04: gzip compression required on all DOTW requests (certification test 12)
'Accept-Encoding' => 'gzip',
```
