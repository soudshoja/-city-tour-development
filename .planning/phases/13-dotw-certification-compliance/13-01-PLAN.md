<objective>
Fix two production bugs in the DOTW booking pipeline:

1. Passenger name sanitization (COMPLY-01): `buildPassengersXml()` currently only calls `htmlspecialchars()`. DOTW requires names to be 2-25 chars, no whitespace, no special characters, multi-word names merged. A new `sanitizePassengerName()` private method applies these rules before XML insertion.

2. Nationality/residence missing from confirmbooking XML (COMPLY-02): `buildConfirmRoomsXml()` does NOT emit `<passengerNationality>` or `<passengerCountryOfResidence>` even though the XSD requires them. `DotwCreatePreBooking` validates these fields but silently drops them when building `$confirmParams`. Both bugs must be fixed together.

Purpose: These are real production bugs — bookings submitted without nationality/residence may be rejected by DOTW. Name sanitization prevents XSD validation failures on names with spaces or special chars.
Output: Updated `DotwService.php` and `DotwCreatePreBooking.php` — no schema changes, no new files.
</objective>
