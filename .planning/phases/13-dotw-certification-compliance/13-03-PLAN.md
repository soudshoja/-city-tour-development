**Fix Test 10 (passenger name sanitization display):**

Test 10's test cases at lines 1049-1056 must accurately reflect Plan 13-01's `sanitizePassengerName()` behavior. The test calls `$this->validatePassengerName()` which checks if a name is already-valid (no sanitization). Update Test 10 to demonstrate both the sanitization and validation steps:

Replace the test cases block to accurately represent the production behavior. The test should:
1. Show that "James Lee" → strips spaces → "JamesLee" → 8 chars → VALID (per COMPLY-01)
2. Show that "J" → strips nothing → "J" → 1 char → INVALID (too short)
3. Show that "O'Brien" → strips `'` → "OBrien" → 6 chars → VALID after sanitization
4. Show that "JohnAlexanderMaximilian123" → strips digits → "JohnAlexanderMaximilian" → 23 chars → VALID (truncation only if >25)
5. A completely invalid case: "123" → strips digits → "" → 0 chars → INVALID (empty after sanitization)

Add a private helper `demonstrateSanitization(string $input): array` that returns `['sanitized' => string, 'valid' => bool]` by:
- Removing whitespace: `preg_replace('/\s+/', '', $input)`
- Removing non-alpha: `preg_replace('/[^A-Za-z]/', '', $result)`
- Checking length: `strlen($result) >= 2 && strlen($result) <= 25`

Update the test loop to call `demonstrateSanitization()` and log the full transformation:
```
✔  'James Lee' → 'JamesLee' (8 chars) — VALID
✔  'J'         → 'J' (1 char)         — INVALID (too short)
```

Keep `endTest(10, true)` at the end (Test 10 is a code demonstration, not a live API call).
