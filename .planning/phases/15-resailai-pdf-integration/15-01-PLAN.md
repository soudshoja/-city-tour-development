**1. processExtraction(array $extractionResult): array**
Entry point. Flow:
a) Extract document_id from `$extractionResult['document_id'] ?? 'unknown'`
b) Call `validateCriticalFields($extractionResult)` — if fails, log error, update DocumentProcessingLog to 'failed', return `['success' => false, 'error' => 'Missing critical field: {field}']`
c) Initialize `$normalizationErrors = []` array to collect non-critical failures
d) Call `normalizeCommonFields($extractionResult, $normalizationErrors)` to get base payload
e) Call `normalizeFinancialFields($extractionResult, $normalizationErrors)` to get financial fields
f) Call the type-specific normalizer based on `$extractionResult['type']`:
   - 'flight' -> `normalizeFlightDetails($extractionResult, $normalizationErrors)`
   - 'hotel' -> `normalizeHotelDetails($extractionResult, $normalizationErrors)`
   - 'visa' -> `normalizeVisaDetails($extractionResult, $normalizationErrors)`
   - 'insurance' -> `normalizeInsuranceDetails($extractionResult, $normalizationErrors)`
g) Merge all normalized parts into one payload
h) If `$normalizationErrors` is non-empty: set `$payload['is_complete'] = false`, set `$payload['enabled'] = false`, log all errors via `logNormalizationErrors($documentId, $normalizationErrors)`
i) Build Request via `buildRequestFromExtraction($payload)`
j) Call `$this->taskWebhook->webhook($request)`
k) If TaskWebhook throws ValidationException, catch it, create the task with is_complete=false via direct Task::create (using the normalized common+financial fields), update DocumentProcessingLog to 'completed' with needs_review=true, return success with warnings
l) Update DocumentProcessingLog to 'completed'
m) Return `['success' => true, 'response' => $response->getData(true)]`

**9. Helper methods:**

`normalizeDate(?string $value, string $field, array &$errors): ?string`
Accept multiple formats and return YYYY-MM-DD HH:MM:SS:
- `DD/MM/YYYY` or `DD/MM/YYYY HH:MM` => parse with Carbon::createFromFormat
- `DD-Mon-YYYY` (e.g., 15-Mar-2026) => Carbon::parse
- `YYYYMMDD` (8 digits) => Carbon::createFromFormat('Ymd')
- `YYYY-MM-DD` or `YYYY-MM-DD HH:MM:SS` => Carbon::parse
- ISO 8601 => Carbon::parse
- If all parsing fails, add `['field' => $field, 'value' => $value, 'error' => 'Unrecognized date format']` to $errors, return null

`normalizeNumeric($value, string $field, array &$errors): ?float`
- If null/empty, return null
- If numeric (int or float), return (float) $value
- If string, strip commas, trim, then try (float) cast
- If not parseable, add to $errors, return null

`normalizeBool($value): bool`
- true, 'true', 'yes', '1', 1 => true
- false, 'false', 'no', '0', 0 => false
- Default: false
