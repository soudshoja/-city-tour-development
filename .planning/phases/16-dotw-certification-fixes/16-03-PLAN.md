**1. Platform Overview**
- Platform name: Soud Laravel — B2B Travel Agency Management Platform
- Platform type: Web-based portal for travel agency agents (B2B, not B2C)
- URL: https://development.citycommerce.group
- Tech stack: Laravel 11, Livewire 3 (reactive UI), Alpine.js, GraphQL (Lighthouse), Tailwind CSS
- Users: Authenticated travel agency agents access via web browser
- Multi-tenant: Company → Branch → Agent hierarchy with isolated data per company

**2. DOTW Integration Architecture**
```
Agent (Browser) → Livewire UI → GraphQL API → DOTW Resolvers → DotwService → DOTW XML API (xmldev.dotwconnect.com)
```
- DotwService makes XML API calls to DOTW V4 endpoints
- Per-company encrypted credentials (CompanyDotwCredential model)
- DotwCacheService for rate caching (150s TTL)
- DotwCircuitBreakerService for transient failure recovery
- DotwAuditService for request/response logging

**3. Booking Flow**
Flow A (Standard): searchHotels → getRooms (browse) → getRooms (blocking with status=checked validation) → confirmBooking
Flow B (APR/Credit Card): searchHotels → getRooms (browse) → getRooms (blocking) → saveBooking → bookItinerary
Cancellation: cancelBooking (confirm=no for charge check) → cancelBooking (confirm=yes with penaltyApplied)

**4. Mandatory Display Features**
For each feature, explain what is displayed and where:

4.1 tariffNotes
- Displayed in room rate detail view after getRooms
- Contains hotel policies, rate notes, supplier terms
- Also included on customer booking voucher

4.2 Cancellation Policies
- Sourced exclusively from getRooms response (not searchHotels)
- Displays: refundable status, cancellation deadline dates, penalty charges per rule
- Handles: cancelRestricted (disables cancel button), amendRestricted (disables modify button)
- Non-refundable rates (APR) clearly marked, cancel/amend UI hidden

4.3 Special Promotions
- specials and specialsApplied elements parsed from getRooms
- Displayed to agent before booking confirmation
- Promotion type and description shown per rate

4.4 Taxes & Fees (propertyFees)
- Parsed from getRooms rateBasis
- includedinprice="Yes" → shown as included in rate
- includedinprice="No" → shown as payable at property
- Fee name, amount, and currency displayed

4.5 Minimum Selling Price (MSP)
- totalMinimumSelling and totalMinimumSellingInRequestedCurrency parsed from getRooms
- Platform enforces selling price >= MSP
- Note: Primary distribution is B2B (travel agencies), not direct B2C

**5. Additional Implementation Details**
- Passenger name sanitization: 2-25 chars, no spaces/special chars, multi-word merged
- Passenger nationality and country of residence: mandatory in all search/getRooms/confirm requests
- Gzip compression: Accept-Encoding: gzip on all requests
- Blocking validation: status=checked required before confirmBooking
- changedOccupancy: adultsCode from validForOccupancy, actualAdults from original search
- Minimum stay: minStay/dateApplyMinStay displayed and enforced
- Allocation details: passed correctly from getRooms blocking to confirm/save
