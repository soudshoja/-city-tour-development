public function findCompanyIdByPhone(string $telephone): ?int
{
    $agent = Agent::where('phone_number', $telephone)
        ->orWhere(DB::raw("CONCAT(country_code, phone_number)"), $telephone)
        ->first();
    if (!$agent) return null;

Build the three core services that every endpoint depends on: phone resolution, hotel import, and fuzzy matching.

**1. PhoneResolverService** (per FOUND-03)
Create `Services/PhoneResolverService.php`. Single public method:
```php
public function resolve(string $phone): ?DotwAIContext
```
Resolution chain (adapted from existing HotelSearchService::findCompanyIdByPhone):
1. Normalize phone: strip `+`, strip leading zeros, keep digits only via `preg_replace('/[^0-9]/', '', $phone)`
2. Try 4 queries against Agent model (same pattern as existing code):
   - `phone_number` = raw phone
   - `phone_number` = normalized phone
   - `CONCAT(country_code, phone_number)` = raw phone
   - `CONCAT(country_code, phone_number)` = normalized phone
3. If no agent found, return null
4. Get `$companyId = $agent->branch?->company_id` -- if null, return null
5. Get `$credentials = CompanyDotwCredential::forCompany($companyId)->first()` -- if null, return null
6. Determine track: if `$credentials->markup_percent > 0` then 'b2c' else 'b2b'
7. Read B2B/B2C enabled from the credentials row first, falling back to config defaults: `$b2bEnabled = $credentials->b2b_enabled ?? config('dotwai.b2b_enabled')`, `$b2cEnabled = $credentials->b2c_enabled ?? config('dotwai.b2c_enabled')`. This allows per-company overrides of the global toggles.
8. Return `new DotwAIContext(agent: $agent, companyId: $companyId, credentials: $credentials, track: $track, markupPercent: $credentials->markup_percent, b2bEnabled: $b2bEnabled, b2cEnabled: $b2cEnabled)`

**2. ResolveDotwAIContext Middleware** (per FOUND-03)
Create `Http/Middleware/ResolveDotwAIContext.php`. Follow the VerifyResailAIToken pattern:
1. Extract `telephone` from request (check JSON body, query params, and header `X-DotwAI-Phone`)
2. If no telephone provided, return DotwAIResponse::error('VALIDATION_ERROR', 'Phone number required', ...)
3. Call `PhoneResolverService::resolve($telephone)`
4. If null, return DotwAIResponse::error('PHONE_NOT_FOUND', ...)
5. If track is disabled (b2b track but b2b not enabled, etc.), return DotwAIResponse::error('TRACK_DISABLED', ...)
6. Attach context: `$request->attributes->set('dotwai_context', $context)`
7. Log: `[DotwAI] Context resolved for {phone} -> company {id}, track {track}`
8. Call `$next($request)`

**3. FuzzyMatcherService** (per FOUND-05)
Create `Services/FuzzyMatcherService.php`. Two public methods:

`findHotels(string $query, ?string $city = null, int $limit = 10): Collection`
1. Normalize query to lowercase, trim
2. **LIKE match first** (fast, uses index): `DotwAIHotel::where('name', 'LIKE', "%{$query}%")`. If city provided, also filter `->where('city', 'LIKE', "%{$city}%")`. Limit to `$limit`. Return if non-empty.
3. **Levenshtein fallback**: Load candidates filtered by city (or limit 500 if no city). Sort by `levenshtein(strtolower($query), strtolower($hotel->name))`. Take top `$limit`. Only return results where distance <= `config('dotwai.fuzzy_match_threshold', 3)`.
4. Return collection (may be empty).

`resolveCity(string $cityName): ?DotwAICity`
1. LIKE match: `DotwAICity::where('name', 'LIKE', "%{$cityName}%")->first()`
2. If no match, Levenshtein on all cities (cities are <5000 records -- safe): `DotwAICity::all()->sortBy(fn($c) => levenshtein(...))->first()`. Return only if distance <= threshold.

`resolveCountry(string $countryName): ?DotwAICountry`
Same pattern as resolveCity but on DotwAICountry, checking both `name` and `nationality_name` columns.
