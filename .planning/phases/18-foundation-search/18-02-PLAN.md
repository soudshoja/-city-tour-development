Build the search orchestration service and message formatter. These are consumed by the controller in Task 2.

**1. HotelSearchService** (core of SRCH-01, SRCH-04, SRCH-05, SRCH-06)

Create `Services/HotelSearchService.php`. Constructor-inject `FuzzyMatcherService` and `DotwService`.

**Method: `searchHotels(DotwAIContext $context, array $input): array`**
Orchestration flow:
1. **Resolve city code**: Use `FuzzyMatcherService::resolveCity($input['city'])`. If no match, try DotwAICity LIKE search. If still null, return error array with code `CITY_NOT_FOUND`.
2. **Resolve nationality code**: Use `FuzzyMatcherService::resolveCountry($input['nationality'] ?? 'Kuwait')`. Fall back to `config('dotwai.default_nationality')`.
3. **Hotel name filter** (if `$input['hotel']` provided per FOUND-05):
   - Call `FuzzyMatcherService::findHotels($input['hotel'], $cityName)` to get local DOTW hotel IDs
   - If matches found, add hotel IDs to DOTW search filters (batches of 50)
   - If no local matches, proceed with city-only search and filter results by name string match after DOTW returns
4. **Build DotwService params**:
   ```php
   $params = [
       'fromDate' => $input['check_in'],   // YYYY-MM-DD
       'toDate' => $input['check_out'],     // YYYY-MM-DD
       'currency' => config('dotwai.default_currency', '520'),
       'rooms' => $this->buildRoomsArray($input['occupancy'], $nationalityCode, $residenceCode),
       'filters' => $this->buildFilters($input),
   ];
   ```
5. **Call DotwService::searchHotels()** with `companyId: $context->companyId`. Wrap in try/catch -- on exception, return error array with code `DOTW_API_ERROR`.
6. **Post-search filtering** (per SRCH-06):
   - Star rating: filter where hotel star_rating matches (if provided)
   - Meal type: filter room types by rateBasis description match (if not "All")
   - Price range: filter by total within priceMin..priceMax (if provided)
   - Refundable only: filter where nonRefundable != 'yes' (if refundable=true)
   - Hotel name: if name provided but not filtered at API level, string match on hotel name
7. **Limit results** to `config('dotwai.search_results_limit', 10)`. Sort by star rating desc, then price asc.
8. **Number results** 1..N for option reference (per SRCH-04).
9. **Cache results** per phone number (per SRCH-04):
   ```php
   Cache::put("dotwai:search:{$normalizedPhone}", $numberedResults, config('dotwai.search_cache_ttl', 600));
   ```
10. Return success array with `hotels` array and metadata (`total_found`, `showing`, `city_name`, `check_in`, `check_out`).

**Helper: `buildRoomsArray(array $occupancy, string $nationalityCode, string $residenceCode): array`** (per SRCH-05)
- Accepts array of `[{adults: int, children_ages: int[]}]` per room
- Maps each to DOTW format: `['no' => 1, 'adultsCode' => $adults, 'children' => $childrenAges, 'passengerNationality' => $natCode, 'passengerCountryOfResidence' => $resCode]`
- Multi-room: each element gets incrementing room number

**Helper: `buildFilters(array $input): array`**
- Build DOTW filter XML elements for star rating, hotel IDs if available

**Method: `getHotelDetails(DotwAIContext $context, string $hotelId, array $input): array`** (per SRCH-02)
1. Build getRooms params with `productId => $hotelId`, dates, currency, occupancy
2. Call `DotwService::getRooms($params, blocking: false, companyId: $context->companyId)` -- browse mode, NO rate blocking (this is Phase 18, browse-only per CONTEXT.md)
3. Parse response: extract room types with name, rateBasisId, rateBasis description (meal type), total price, cancellation rules, specials, tariff notes, isRefundable, isAPR, minStay, MSP
4. Apply B2C markup if track is b2c: `$displayPrice = ceil($price * (1 + $context->markupPercent / 100))`. Enforce MSP: `$displayPrice = max($displayPrice, $minimumSellingPrice)`.
5. Map specials using the dotw-mapping.md promotion type mapping (mapSpecialType logic)
6. Map cancellation rules to structured array (fromDate, charge, chargeType, cancelRestricted)
7. Return structured array with hotel info + rooms array

**Method: `getCities(string $countryName): array`** (per SRCH-03)
1. Resolve country name to DOTW country code via FuzzyMatcherService::resolveCountry
2. If local dotwai_cities has data for this country code, return from local table (fast path)
3. If no local data, call DotwService::getCityList($countryCode), and upsert results into dotwai_cities for future use
4. Return array of `[{code, name, country_code}]`

**Method: `getCachedResults(string $phone): ?array`** (supports SRCH-04)
Return `Cache::get("dotwai:search:{$normalizedPhone}")`.

**1. SearchController**
Create `Http/Controllers/SearchController.php`. Namespace: `App\Modules\DotwAI\Http\Controllers`.
Extend `Illuminate\Routing\Controller` (not the app's base Controller -- module isolation).
Constructor-inject `HotelSearchService` and `MessageBuilderService`.

**Method: `searchHotels(SearchHotelsRequest $request): JsonResponse`** (per SRCH-01)
1. Get context: `$context = $request->attributes->get('dotwai_context')` -- this was set by middleware
2. Call `$result = $this->searchService->searchHotels($context, $request->validated())`
3. If result has error: return `DotwAIResponse::error($result['code'], $result['message'], MessageBuilderService::formatError($result['code']), $result['suggestedAction'] ?? null)`
4. Format WhatsApp message: `$whatsapp = MessageBuilderService::formatSearchResults($result['hotels'], config('dotwai.display_currency'))`
5. Build options: `$options = MessageBuilderService::buildWhatsAppOptions('search')`
6. Return `DotwAIResponse::success($result, $whatsapp, $options)`

**Method: `getHotelDetails(GetHotelDetailsRequest $request): JsonResponse`** (per SRCH-02)
1. Get context from request attributes
2. Call `$result = $this->searchService->getHotelDetails($context, $request->input('hotel_id'), $request->validated())`
3. If error: return DotwAIResponse::error(...)
4. Format WhatsApp message: `$whatsapp = MessageBuilderService::formatHotelDetails($result['hotel'], $result['rooms'], config('dotwai.display_currency'))`
5. Build options: `$options = MessageBuilderService::buildWhatsAppOptions('details')`
6. Return `DotwAIResponse::success($result, $whatsapp, $options)`

**Method: `getCities(Request $request): JsonResponse`** (per SRCH-03)
1. Validate: `country` parameter required (string, min:2)
2. Get context from request attributes (for company-specific DOTW credentials)
3. Call `$result = $this->searchService->getCities($request->input('country'))`
4. If error or empty: return DotwAIResponse::error('CITY_NOT_FOUND', ...)
5. Format WhatsApp message: `$whatsapp = MessageBuilderService::formatCityList($result)`
6. Return `DotwAIResponse::success(['cities' => $result], $whatsapp)`
