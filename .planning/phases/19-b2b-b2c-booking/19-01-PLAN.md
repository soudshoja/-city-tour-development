    **3. Create BookingService** at `app/Modules/DotwAI/Services/BookingService.php`:
    Namespace: `App\Modules\DotwAI\Services`

    Constructor: inject `CreditService $creditService` and `HotelSearchService $searchService` (from Phase 18).

    **Method `prebook(DotwAIContext $context, array $input): array`:**
    1. Resolve the hotel and room from cached search results or direct input:
       - If `option_number` provided: load from `Cache::get('dotwai:search:' . $this->normalizePhone($input['telephone']))`, pick the option at index `option_number - 1`. Extract the first room's details (room_type_code, rate_basis_id, allocation_details, etc.).
       - If `hotel_id` provided directly: requires room_type_code and rate_basis_id in input. Call DotwService getRooms with blocking=true to get fresh allocation.
    2. Build blocking params for DotwService::getRooms:
       ```php
       $dotwService = new DotwService($context->companyId);
       $blockResult = $dotwService->getRooms([
           'fromDate' => $input['check_in'],
           'toDate' => $input['check_out'],
           'currency' => config('dotwai.default_currency'),
           'productId' => $hotelId,
           'rooms' => $this->buildRoomSelections($roomData, $input['occupancy']),
       ], true, null, null, $context->companyId);
       ```
    3. Verify the blocking succeeded (check `$blockResult` has allocation data and status 'checked').
    4. If B2C: enforce MSP -- ensure display_total_fare >= minimumSellingPrice. If the marked-up price is below MSP, use MSP as the display price (per B2C-05).
    5. Determine track: `$context->isB2B()` with credit check -> 'b2b', `$context->isB2B()` without credit -> 'b2b_gateway', `$context->isB2C()` -> 'b2c'.
    6. Create DotwAIBooking record with status='prebooked'. Store ALL rate details needed for confirm: allocation_details, room_type_code, rate_basis_id, nationality_code, residence_code, changed_occupancy, rooms_data (occupancy breakdown per room), cancellation_rules, original_total_fare, display_total_fare with markup, etc.
    7. Extract cancellation_deadline from the cancellation_rules (earliest `fromDate` where charge > 0).
    8. Return array: `['prebook_key' => ..., 'hotel_name' => ..., 'check_in' => ..., 'check_out' => ..., 'display_total_fare' => ..., 'original_total_fare' => ..., 'currency' => ..., 'is_refundable' => ..., 'is_apr' => ..., 'cancellation_rules' => ..., 'cancellation_deadline' => ..., 'track' => ..., 'needs_payment' => bool]`
    9. `needs_payment` is true for 'b2c' and 'b2b_gateway' tracks, false for 'b2b' (credit line).
    10. Return `['error' => true, 'code' => 'RATE_UNAVAILABLE', ...]` if blocking fails.

    Helper `normalizePhone(string $phone): string`: strip +, spaces, leading zeros. Same pattern as PhoneResolverService.

    Helper `buildRoomSelections(array $roomData, array $occupancy): array`: Build the room selections array for getRooms blocking call. Each room needs adultsCode, childrenAges, rateBasisId, roomTypeCode, allocationDetails, nationality, residence.

    **Method `confirmWithCredit(DotwAIBooking $booking, DotwAIContext $context, array $passengers, ?string $email): array`:**
    1. Idempotency gate: if `$booking->confirmation_no` is set, return existing confirmation (per Pitfall 3 from RESEARCH).
    2. Get clientId via `$this->creditService->getClientIdForCompany($context->companyId)`.
    3. Call `$this->creditService->checkAndDeductCredit($clientId, $context->companyId, $booking->original_total_fare, $booking->prebook_key)`.
    4. If insufficient credit, return `['error' => true, 'code' => 'INSUFFICIENT_CREDIT', 'available' => balance]`.
    5. Call DOTW confirmBooking (or saveBooking + bookItinerary for APR rates per locked decision):
       ```php
       $dotwService = new DotwService($context->companyId);
       if ($booking->is_apr) {
           $itinerary = $dotwService->saveBooking($confirmParams, null, null, $context->companyId);
           $confirmation = $dotwService->bookItinerary($itinerary['bookingCode'], null, null, $context->companyId);
       } else {
           $confirmation = $dotwService->confirmBooking($confirmParams, null, null, $context->companyId);
       }
       ```
    6. Build $confirmParams using data stored on the booking: productId, fromDate, toDate, currency (original_currency), sendCommunicationTo (email), customerReference ('DOTWAI-' . $booking->id), rooms (with passenger details, sanitized via DotwService::sanitizePassengerName, allocationDetails, roomTypeCode, rateBasisId, adultsCode, actualAdults, children, nationality, residence).
    7. If DOTW fails: refund credit via `$this->creditService->refundCredit()`, update booking status to 'failed', return error.
    8. On success: update booking with confirmation_no, booking_ref, payment_guaranteed_by, status='confirmed', payment_status='credit_applied'. Store guest_details.
    9. Return confirmation data array.

    **Method `confirmAfterPayment(DotwAIBooking $booking): array`:**
    This is the method called by Plan 02's ConfirmBookingAfterPaymentJob. It:
    1. Idempotency gate.
    2. Re-blocks the rate (critical per CONTEXT.md locked decision): calls DotwService::getRooms with blocking=true using stored hotel_id, dates, room details. Updates allocation_details with fresh token.
    3. If re-block fails: return `['error' => true, 'code' => 'RATE_UNAVAILABLE', 'needs_refund' => true]`.
    4. If re-block succeeds: build confirmParams with fresh allocation, call confirmBooking (or saveBooking+bookItinerary for APR).
    5. On success: update booking with confirmation_no, status='confirmed'. Return confirmation.
    6. On DOTW failure: update booking status='failed'. Return error with needs_refund=true.

    **Method `getCompanyBalance(DotwAIContext $context): array`:**
    1. Get clientId via CreditService.
    2. Return `$this->creditService->getBalance($clientId)`.
