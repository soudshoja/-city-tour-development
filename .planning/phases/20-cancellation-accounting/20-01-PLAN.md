1. Add `STATUS_CANCELLATION_PENDING = 'cancellation_pending'` constant to DotwAIBooking model (after STATUS_EXPIRED line). This is a string column, no migration needed.

2. Create `app/Modules/DotwAI/Services/CancellationService.php` — standalone class (NOT extending BookingService). Constructor takes `AccountingService $accountingService` and `CreditService $creditService`.

   Main method: `cancel(DotwAIContext $context, array $input): array`
   - $input keys: prebook_key, confirm ('no'|'yes'), penalty_amount (required when confirm=yes)
   - Resolve booking: `DotwAIBooking::where('prebook_key', $input['prebook_key'])->where('company_id', $context->companyId)->first()`
   - Validate status: must be STATUS_CONFIRMED or STATUS_CANCELLATION_PENDING, otherwise return `DotwAIResponse::error('CANCELLATION_NOT_ALLOWED')`
   - Validate booking_ref is not null (cannot cancel if never confirmed with DOTW)

   Step 1 (confirm=no): Call `$dotwService = new DotwService($context->companyId); $result = $dotwService->cancelBooking(['confirm' => 'no', 'bookingCode' => $booking->booking_ref])`. Update booking status to STATUS_CANCELLATION_PENDING. Return preview data with charge/refund amounts and WhatsApp message via `MessageBuilderService::formatCancellationPending()`.

   Step 2 (confirm=yes): Call DotwService::cancelBooking with confirm=yes and penaltyApplied. IMPORTANT: Call DOTW API FIRST, THEN open DB::transaction for Eloquent writes (DOTW HTTP cannot be rolled back). Inside the transaction:
   - Update booking status to STATUS_CANCELLED
   - If (float)$result['charge'] > 0: call $this->accountingService->createCancellationEntries($booking, (float)$result['charge'], $context)
   - If booking track is TRACK_B2B: call $this->creditService->refundCredit() with amount = display_total_fare - charge (if positive)
   Return confirmed data with WhatsApp message via `MessageBuilderService::formatCancellationConfirmed()`.

   Error handling: Wrap DotwService calls in try/catch. On exception, log to 'dotw' channel and return DotwAIResponse::error('DOTW_API_ERROR'). If DOTW cancel succeeds but DB::transaction fails, log critical error with prebook_key + DOTW response, set booking status to 'cancelled' outside transaction (best-effort), and return success to user (cancellation did happen on DOTW side).

3. Create `app/Modules/DotwAI/Services/AccountingService.php` — standalone class. Constructor takes no dependencies (uses static model calls).

   Main method: `createCancellationEntries(DotwAIBooking $booking, float $penaltyAmount, DotwAIContext $context): void`
   - Resolve clientId: `CreditService::getClientIdForCompany($context->companyId)` — throw RuntimeException if null
   - Lookup accounts with `Account::withoutGlobalScopes()->where('company_id', $context->companyId)->where('name', 'LIKE', '%Client%')->first()` for receivable. Use `->where('name', 'LIKE', '%Revenue%')` for revenue. Log warning and skip JournalEntry creation (but still create invoice) if accounts not found — do NOT throw.
   - Create Invoice: client_id from resolved clientId, agent_id from $context->agent->id, currency from $booking->display_currency, sub_amount and amount = $penaltyAmount, status = InvoiceStatus::UNPAID->value, invoice_date = now()->toDateString(), due_date = now()->toDateString(), label = 'Cancellation Penalty: ' . $booking->prebook_key
   - Create InvoiceDetail: invoice_id, invoice_number from $invoice->invoice_number, task_description = 'Hotel cancellation penalty - ' . $booking->hotel_name, task_price = $penaltyAmount, supplier_price = $penaltyAmount
   - Create 2 JournalEntry records (debit receivable, credit revenue): company_id = $context->companyId, branch_id = $context->agent->branch_id, invoice_id = $invoice->id, transaction_date = now(), currency = $booking->display_currency, type = 'cancellation'. ALL JournalEntry::create() calls MUST include explicit company_id (ACCT-04).
   - For B2B bookings where penalty was deducted from credit line: update invoice status to InvoiceStatus::PAID->value

4. Add to MessageBuilderService (same static pattern as existing formatters):
   - `formatCancellationPending(array $data): string` — $data keys: hotel_name, check_in, check_out, penalty_amount, currency, booking_ref, refund_amount. Bilingual AR/EN, shows penalty amount, asks for explicit confirmation to proceed.
   - `formatCancellationConfirmed(array $data): string` — $data keys: hotel_name, booking_ref, penalty_amount, currency, is_free_cancellation. Bilingual AR/EN. MUST include warning text: "Please note: DOTW cancellation confirmation may take additional time to reflect on the portal" / "يرجى العلم: قد يستغرق تأكيد الإلغاء من DOTW وقتاً إضافياً للظهور على البوابة" (CANC-02 locked decision).

5. Add to DotwAIResponse: new error code constants `CANCELLATION_NOT_ALLOWED` and `CANCELLATION_FAILED` with bilingual default messages. Follow existing pattern (const + DEFAULT_MESSAGES entry).
