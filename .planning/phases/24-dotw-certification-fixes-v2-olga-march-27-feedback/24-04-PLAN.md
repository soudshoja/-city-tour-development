    **CERT-08: B2B/B2C Connection Document**

    Create `docs/DOTW-B2B-B2C-Connection-Guide.md` with these sections:

    **1. System Overview**
    - City Commerce Group operates a travel agency management platform
    - DOTW hotel booking is integrated as a module within the platform
    - All DOTW API calls go through the platform's backend (no direct client-to-DOTW communication)

    **2. Multi-Tenant Architecture**
    - Company -> Branch -> Agent hierarchy
    - Each company stores its own DOTW credentials (username, password, company ID) in company settings
    - Data isolation: each company sees only its own bookings, agents, and transactions
    - Companies are onboarded by the platform admin who configures their DOTW credentials

    **3. Client Interface: WhatsApp**
    - End users (agents and customers) interact via WhatsApp
    - WhatsApp Business API (Resayil) handles message routing
    - AI agent (powered by n8n workflow with LLM) understands natural language requests
    - The AI agent translates user messages into structured API calls to the platform

    **4. B2B Flow (Agency Agents)**
    - Agent sends WhatsApp message: "Search hotels in Dubai, 2 adults, March 15-18"
    - AI agent calls platform REST API -> platform calls DOTW XML API
    - Results returned to agent via WhatsApp with option numbers
    - Agent selects hotel and room, provides passenger details
    - Booking modes:
      a. Credit Line: agent's company has pre-approved credit -> booking confirmed immediately, credit deducted
      b. Payment Link: no credit line -> payment link sent via WhatsApp -> after payment, booking confirmed with DOTW
    - Voucher delivered via WhatsApp

    **5. B2C Flow (Direct Customers)**
    - Customer interacts via WhatsApp (same interface)
    - Prices include configurable markup (default 20%, enforced MSP)
    - Customer receives payment link after prebook
    - After payment: system re-blocks rate with DOTW, confirms booking, sends voucher
    - Company receives revenue with markup applied

    **6. How Other Agencies Connect**
    - Agency contacts City Commerce Group for onboarding
    - Admin creates Company record with agency's DOTW credentials
    - Agency's WhatsApp number is linked to their company profile
    - Their agents and customers get isolated WhatsApp-based booking access
    - Each agency can have custom markup percentages and payment gateway settings
    - White-label capability: responses branded per agency

    **7. Technical Architecture**
    ```
    WhatsApp (Resayil) -> n8n AI Agent -> REST API -> Laravel Backend -> DOTW XML API
    ```
    - REST endpoints: /api/dotwai/agent-b2b, /api/dotwai/agent-b2c
    - Authentication: API token per device/agent
    - Session management: per-phone cache with 60-minute TTL
    - All mandatory DOTW features displayed in WhatsApp messages

    **8. Data Flow**
    - Search: user request -> DOTW searchhotels -> filtered results -> WhatsApp
    - Prebook: user selects room -> DOTW getRooms (blocking) -> rate locked -> prebook stored
    - Book: payment confirmed -> DOTW confirmBooking -> booking stored -> voucher sent
    - Cancel: user requests -> DOTW cancelbooking (2-step) -> penalty shown -> confirmed

    **9. Security**
    - DOTW credentials encrypted at rest in company settings
    - All API calls authenticated via bearer token
    - Rate limiting on all endpoints
    - Audit trail for every booking action

    Format as a professional document suitable for sending to Olga/DOTW team.
