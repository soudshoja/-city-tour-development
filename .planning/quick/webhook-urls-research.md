# Webhook URLs Research - Laravel-N8N Integration

**Researched:** 2026-03-10
**Domain:** Webhook Configuration, N8N Integration, Production Deployment
**Confidence:** HIGH

## Summary

This research documents the actual production webhook URLs for Laravel-N8N integration, the environment configuration method, and how to configure for new suppliers (e.g., Flydubai).

**Key Finding:** The production N8N webhook URL is **NOT configured** in `.env.example` (empty placeholder). The production Laravel callback URL is `https://development.citycommerce.group/api/webhooks/n8n/extraction`.

**Primary Recommendation:** Configure `N8N_WEBHOOK_URL` in production environment to point to the N8N instance, and ensure `N8N_WEBHOOK_SECRET` matches between Laravel and N8N credentials.

## Environment Configuration

### Required Environment Variables

```bash
# .env - N8N Integration
N8N_WEBHOOK_URL=https://your-n8n-instance.com/webhook/supplier-document-processing
N8N_WEBHOOK_SECRET=your-shared-hmac-secret-here

# Optional - Alternative configuration
N8N_BASE_URL=https://your-n8n-instance.com
N8N_WEBHOOK_PATH=/webhook/supplier-document-processing
N8N_API_KEY=your-n8n-api-key
```

### Configuration Files

| File | Variable | Purpose |
|------|----------|---------|
| `config/services.php` | `n8n.webhook_url` | Primary webhook URL (env: `N8N_WEBHOOK_URL`) |
| `config/services.php` | `n8n.webhook_secret` | HMAC secret for signature verification (env: `N8N_WEBHOOK_SECRET`) |
| `config/webhook.php` | `n8n.base_url` | N8N base URL (env: `N8N_BASE_URL`, default: `http://localhost:5678`) |
| `config/webhook.php` | `n8n.webhook_path` | Webhook path suffix (env: `N8N_WEBHOOK_PATH`, default: `/webhook/document-processing`) |

**Note:** `config/services.php` is the primary source for the webhook URL used by `DocumentProcessingController`:

```php
// config/services.php line 109-112
'n8n' => [
    'webhook_url' => env('N8N_WEBHOOK_URL'),
    'webhook_secret' => env('N8N_WEBHOOK_SECRET', 'default-secret'),
],
```

## How to Configure for New Supplier (Flydubai)

### Step 1: Ensure N8N Webhook URL is Configured

```bash
# Add to .env
N8N_WEBHOOK_URL=https://your-n8n-instance.com/webhook/supplier-document-processing
N8N_WEBHOOK_SECRET=your-secure-hmac-secret-min-32-chars
```

### Step 2: Configure N8N Credentials

In N8N UI:
1. Navigate to **Credentials** -> **Add Credential**
2. Type: **Generic Credential**
3. Name: `Laravel Webhook Secret`
4. Add field:
   - Key: `secret`
   - Value: Same as `N8N_WEBHOOK_SECRET` in Laravel `.env`
5. Save

### Step 3: Verify Supplier Routing in N8N Workflow

The N8N workflow routes by `supplier_id`:

| Supplier ID | Supplier Name | Processing Type |
|-------------|---------------|-----------------|
| 1 | Jazeera Airways | AIR (deferred) |
| 2 | Flydubai | AIR (deferred) |
| 3 | ETA UK | PDF (Tika) |
| 4 | The Skyrooms | PDF (Tika) |
| 5 | Air Arabia | AIR (deferred) |
| 6 | Indigo | AIR (deferred) |
| 7 | Cham Wings | AIR (deferred) |
| 8 | VFS Global | PDF (Tika) |

## Security Requirements

### HMAC Signature Verification

Both directions use HMAC-SHA256 signing:

**Laravel to N8N (Outbound):**
```php
// In DocumentProcessingController
$timestamp = now()->timestamp;
$hmac = hash_hmac('sha256', json_encode($payload), $webhookSecret);
// Headers: X-Signature, X-Timestamp
```

**N8N to Laravel (Callback):**
```javascript
// In N8N "Compute Callback HMAC" node
const crypto = require('crypto');
const hmac = crypto.createHmac('sha256', webhookSecret)
  .update(body)
  .digest('hex');
// Header: X-Signature
```

### Configuration Checklist

- [ ] `N8N_WEBHOOK_URL` set in production `.env`
- [ ] `N8N_WEBHOOK_SECRET` matches in Laravel and N8N
- [ ] `APP_URL` set correctly for callback URL generation
- [ ] N8N can reach Laravel callback URL (firewall rules)
- [ ] Laravel can reach N8N webhook URL (network connectivity)

## Common Issues and Solutions

### Issue 1: N8N_WEBHOOK_URL Not Set

**Symptom:** `ERR_N8N_CONFIG_MISSING` error, 500 response

**Solution:**
```php
// DocumentProcessingController.php lines 66-76
if (!$n8nWebhookUrl) {
    return response()->json([
        'error' => 'N8n configuration missing — set N8N_WEBHOOK_URL in .env',
    ], 500);
}
```

### Issue 2: Callback URL Not Accessible from N8N

**Symptom:** N8N execution shows connection timeout, Laravel never receives callback

**Solution:** Ensure `APP_URL` is set to accessible hostname:
```bash
# Production
APP_URL=https://development.citycommerce.group

# For development behind firewall, use ngrok or similar
APP_URL=https://your-ngrok-subdomain.ngrok.io
```

### Issue 3: HMAC Signature Mismatch

**Symptom:** `ERR_HMAC_INVALID` in N8N or Laravel rejects callback

**Solution:**
1. Verify `N8N_WEBHOOK_SECRET` in Laravel `.env`
2. Verify N8N credential "Laravel Webhook Secret" has same value
3. Both values must match exactly (case-sensitive)
